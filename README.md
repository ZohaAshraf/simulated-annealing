# 🗺️ TSP Simulated Annealing

Solving the **Travelling Salesman Problem (TSP)** using **Simulated Annealing** in Python — with a comparison of geometric and linear cooling schedules.

---

## 📌 Overview

Given a set of 10 cities on a 2D plane, the goal is to find the shortest possible route that visits every city exactly once and returns to the starting point.

This project implements **Simulated Annealing (SA)**, a probabilistic metaheuristic optimization algorithm, and compares two temperature cooling strategies:

- **Geometric cooling** — temperature decays exponentially each iteration
- **Linear cooling** — temperature decreases at a fixed rate each iteration

## ⚙️ How It Works

### Simulated Annealing

At each iteration:
1. A **neighbour** solution is generated by swapping two random cities in the current route
2. If the neighbour is **better**, it is always accepted
3. If it is **worse**, it is accepted with probability `e^(-Δ/T)` — this allows escaping local optima
4. Temperature `T` gradually decreases, reducing the chance of accepting worse solutions over time

### Cooling Schedules

| Schedule | Formula |
|----------|---------|
| Geometric | `T = T₀ × αⁱ` (default: T₀=1000, α=0.995) |
| Linear | `T = T₀ - (T₀ / iterations) × i` (default: T₀=1000, 10,000 steps) |

## 🚀 Getting Started

### Prerequisites

```bash
pip install matplotlib
```

### Run

```bash
python tsp_sa.py
```

This will:
- Run SA with both cooling schedules
- Print results to the console
- Save a convergence plot as `sa_cost_curve.png`

## 🛠️ Parameters

You can tweak the following in `simulated_annealing()`:

| Parameter | Default | Description |
|-----------|---------|-------------|
| `T0` | 1000 | Initial temperature |
| `alpha` | 0.995 | Cooling rate (geometric only) |
| `iterations` | 10000 | Number of iterations |
| `schedule` | `'geometric'` | `'geometric'` or `'linear'` |
